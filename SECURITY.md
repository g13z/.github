# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| latest  | :white_check_mark: |
| < latest | :x:               |

## Reporting a Vulnerability

We use GitHub's private vulnerability reporting to handle security disclosures.

**Please do not open a public issue for security vulnerabilities.**

To report a vulnerability:
1. Go to the affected repository on GitHub
2. Click the **Security** tab
3. Click **Report a vulnerability**
4. Fill out the advisory form

We will acknowledge receipt within **48 hours** and aim to provide a full response 
within **7 days**, including an assessment and expected remediation timeline.

If you are unsure whether an issue is a security vulnerability, report it privately 
and we will triage it accordingly.

## Disclosure Policy

We follow coordinated disclosure. Once a fix is available and deployed, we will 
publish a GitHub Security Advisory crediting the reporter (unless they prefer 
to remain anonymous). We DO NOT pay out monetray rewards.

## Scope

The following are in scope:
- Code in repositories under this organization
- Published packages maintained by this organization

The following are out of scope:
- Issues in dependencies (report those upstream)
- Social engineering attacks
- Denial of service
